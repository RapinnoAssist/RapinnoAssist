GenAI

In the Knowledge Base module, select GenAI to define knowledge on key domain terms, including business, products, and services.

Gen AI part is primarily used to connect data from various sources to train chatbots. It processes static data, eliminating the need for manual training by automating the training process for chatbots.

Feeding Knowledge to the Chatbot​

There are multiple ways to provide knowledge to the chatbot:

  1. Website
  2. Files
  3. Jira
  4. GitHub
  5. Confluence (and 30+ more connectors)

Website​

You can train the chatbot using a website link.

Using a domain name, the chatbot can be trained on all URLs that belong to the domain. To delete a specific URL from the domain, use the bin icon. To delete the entire domain, use the bin icon.

An exact URL can be used to train the bot for that specific URL.

Files​

You can train the bot using PDF files.

Jira​

Feed data to the bot from Jira by providing the API Token, Username, and Instance URL.

GitHub​

Feed data to the bot from GitHub by providing the Repository URL, Branch, and Personal Access Token.

Confluence​

Feed data to the bot from Confluence by providing the Confluence API Token, Confluence Username, Confluence Instance URL, and Confluence Space.
